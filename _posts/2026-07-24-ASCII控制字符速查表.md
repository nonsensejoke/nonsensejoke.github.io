在终端里按 **Ctrl + 某个键**，其实是在输入一个 ASCII 控制字符。ASCII 表最前面的 32 个码位（0–31）外加 127，都是不可打印的"控制字符"，它们各自触发不同的行为：有的是发信号（`Ctrl-C` 中断），有的是行编辑（`Ctrl-A` 移到行首），有的干脆就等于某个普通按键（`Ctrl-I` 就是 Tab）。

这份速查表按**使用场景**把它们分组整理，方便快速查阅。内容整理自 Julia Evans（[jvns.ca/ascii](https://jvns.ca/ascii)）的同名速查表，她还写过一篇[充满注意事项的详解博文](https://jvns.ca/blog/2024/10/31/ascii-control-characters/)，想深挖细节的推荐一读。

需要提醒的是：具体行为**取决于你所处的程序**。同一个 `Ctrl-D`，在 shell 里是 EOF（退出），在命令行中间又是删除右侧字符。下面标注的是最常见的默认行为。

## 📖 分类详解

### 1. 信号与进程控制（由终端驱动处理）
这几个是发给内核终端驱动的，不属于 readline，几乎在任何前台程序里都生效：

*   **`Ctrl-C`**：发送 `SIGINT`，中断当前前台进程（最常用的"停下来"）。
*   **`Ctrl-Z`**：发送 `SIGTSTP`，把当前进程挂起到后台，之后可用 `fg` 恢复。
*   **`Ctrl-\`**：发送 `SIGQUIT`，比 `Ctrl-C` 更"暴力"，通常还会生成 core dump。
*   **`Ctrl-D`**：在行首按下代表 EOF（输入结束），在 shell 里等于退出；在一行中间则是删除光标右侧字符。

### 2. 命令行编辑（readline / emacs 风格）
在 bash、zsh 等使用 readline 的环境中生效：

*   **`Ctrl-A`** / **`Ctrl-E`**：跳到行首 / 行尾。
*   **`Ctrl-B`** / **`Ctrl-F`**：光标左移 / 右移一个字符。
*   **`Ctrl-U`**：删除光标前的整段（到行首）。
*   **`Ctrl-K`**：删除光标到行尾的内容。
*   **`Ctrl-W`**：删除光标前的一个单词。
*   **`Ctrl-Y`**：粘贴（yank）刚才用 `Ctrl-U/K/W` 删掉的内容。
*   **`Ctrl-_`**：撤销上一次编辑。

### 3. 历史与屏幕控制
*   **`Ctrl-R`**：反向增量搜索历史命令（超实用）。
*   **`Ctrl-P`** / **`Ctrl-N`**：上一条 / 下一条历史命令（等价于上下方向键）。
*   **`Ctrl-L`**：清屏（等价于 `clear`）。
*   **`Ctrl-S`** / **`Ctrl-Q`**：暂停 / 恢复终端输出（软件流控 XOFF/XON）。
*   **`Ctrl-G`**：响铃 🔔。

### 4. 等价于普通按键
这些控制字符本质上就是某个常见键，历史上很多键盘正是用它们来实现的：

*   **`Ctrl-I`** = **Tab**
*   **`Ctrl-M`** = **Enter / 回车**（`\r`，回车符）
*   **`Ctrl-J`** = **换行**（`\n`，换行符）
*   **`Ctrl-[`** = **ESC**
*   **`Ctrl-H`** 与 **`Ctrl-?`** = **Backspace**（两种退格的历史遗留）

### 5. 其他 / 少用
*   **`Ctrl-V`**：转义下一个字符，用于原样输入一个控制字符（如输入真正的 Tab）。
*   **`Ctrl-T`**：在 BSD 系统上发送 `SIGINFO`（显示进程状态）；在 readline 里则是交换光标前两个字符。
*   **`Ctrl-X`**：emacs 的前缀键，后接其他键组成组合命令。
*   **`Ctrl-]`**：退出 telnet 会话回到命令提示符。
*   **`Ctrl-@`**（NUL）、**`Ctrl-O`**、**`Ctrl-^`**：一般没有常用默认功能。

## ⚠️ 几个常见陷阱

1.  **终端卡住不动？** 多半是手滑按了 **`Ctrl-S`**（暂停输出），按 **`Ctrl-Q`** 即可恢复——很多人以为是死机了。
2.  **行为随程序而变**：同一个组合键在 shell、vim、emacs、telnet 里含义可能完全不同。上表标注的是最常见的默认场景。
3.  **`Ctrl-H` vs `Ctrl-?`**：两者都可能是退格，具体哪个生效取决于终端的 `stty erase` 设置，这也是有时按退格却打出 `^H` 的原因。
4.  **`Ctrl-C` 不一定管用**：它只是发信号，程序可以选择忽略 `SIGINT`；此时可试 `Ctrl-\`（SIGQUIT）。
